Important note here: as practice shows “The rails way” does not work for scallable apps. So in production code i would recommend to add another layer of indirection and query a @project via repository pattern. I know that it is not “the rails way”, but you can cover your ass and leverage best sides of duck typing, by easily replacing implementation of repository later on if you need to (like you can easily add caching support, change persistence layer without changing you client code, etc.)
